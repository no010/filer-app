# filer 使用手册

> 监控下载目录 → 自动分析预填建议元信息 → 收件箱确认（重复下载给提示）→ 整理分发到单一根目录下的结构化子文件夹。本地 SQLite 索引，文件在本地移动，不上云。

---

## 1. 安装与首次启动

双击安装包 → 开始菜单出现 "filer"。首次打开检测到无 `watch_dir`/`dest_root` → 弹出强制设置向导（不可关闭直到保存）：

```
┌───────────────────────────────────────────────┐
│ 首次启动配置                                   │
│ ⚠ 需配置下载监听目录 + 归档根目录后才能进入   │
├───────────────────────────────────────────────┤
│ 下载监听目录  [C:\Users\You\Downloads    ] 选择│
│ 归档根目录    [D:\Filer                  ] 选择│
│   规则子路径会拼到此目录下，例如              │
│   D:\Filer\Datasheets\ST\、D:\Filer\Receipts\..│
│ 时区          [系统时区            ] 常用…     │
│ 冲突策略      [rename（加后缀）  ▼]             │
│ 默认动作      [move（移动）      ▼]             │
│ 署名          [你的名字                       ]│
│ ☑ 开机自动启动                                 │
│ ☐ 关闭时最小化到系统托盘                       │
│ 规则集（只读，按顺序匹配）                     │
│   ...                                          │
│                          [保存并进入应用]      │
└───────────────────────────────────────────────┘
```

保存即进入主界面。配置文件：`%APPDATA%\filer\config.toml`（明文，可手改）。

> **关闭时最小化到系统托盘**：首次点窗口关闭按钮时弹一次原生询问（默认「是」= 最小化到托盘，「否」= 直接退出），选择会被记住、不再问。驻留托盘时后台监听/归档不中断：左键托盘图标或右键「显示 filer」恢复窗口，右键「退出 filer」真正退出。后续可在「设置」里随时改这个开关。

## 2. 日常使用

主界面两个 tab：**收件箱** / **历史**。

### 收件箱

filer 后台监听下载目录。文件下载完成（size 稳定 ≥3s，临时文件 `.part`/`.crdownload` 等忽略）后自动分析入箱：

```
┌──────────────────────────────────────────────────────────────────┐
│ 收件箱 · 3 项待确认                       [立即扫描下载目录]    │
├──────────┬──────────┬──────┬──────┬──────────┬──────┬───────────┤
│ 文件名   │ 类型     │ 大小 │ 分类 │ 建议目标 │ 标签 │ 操作      │
├──────────┼──────────┼──────┼──────┼──────────┼──────┼───────────┤
│STM32F103 │pdf·ST    │1.9MB │Data..│D:\Filer\ │+标签 │[归档][编辑]│
│..pdf     │72 页     │      │sheet │Datasheets│      │[跳过]     │
│发票.pdf  │pdf       │88KB  │Recei│D:\Filer\ │      │           │
│          │          │      │pt   │Receipts\ │      │           │
└──────────┴──────────┴──────┴──────┴──────────┴──────┴───────────┘
```

- **归档**：按建议直接移动到目标（move，冲突按策略处理）。
- **编辑**：打开 ReviewModal，改分类/目标目录/文件名/动作(move/copy)/标签，实时预览最终路径后归档。
- **跳过**：标为 ignored，不移走。
- **+标签**：行内输入回车快速加标签 chip，× 删除。
- **立即扫描**：手动回填下载目录里已有但未入箱的文件。

### 重复下载提示

filer 在文件入箱时就算 sha256，若与某条已归档记录内容相同，该行目标列显示「⚠ 与 #N 内容相同」，操作列换成 4 个处理按钮：

| 按钮 | 行为 | DB 记录 |
|---|---|---|
| **跳过** | 保留已归档的旧文件，新文件留在下载目录，本条标 ignored | `dedup_decision=skip` |
| **保留两者** | 新文件照常归档（强制 rename 加后缀），两份都留 | `keep_both` |
| **替换** | 删除旧归档文件（旧记录标 replaced），新文件归档到自己的建议位置 | `replace` |
| **删除新文件** | 删除新下载的源文件，保留旧归档，本条标 ignored | `delete_new` |

> 内容判重（sha256）必须下载完成后才能算——要完整字节流。所以判重点在「入箱」那一刻，不是下载前。

### 历史

已归档记录，分页加载：

- **打开**：在系统资源管理器里定位到归档文件（Windows 上 `explorer /select`）。
- **撤销**：move 的移回原下载路径；copy 的删除归档副本。记录回到收件箱可重新归档。
- 标签可继续编辑。

### 回顾（长期未用识别）

第三个 tab 按陈旧度排序已归档文件（最久未触达在前），用记忆曲线式分层提示：

| 天数 | 标签 |
|---|---|
| <30 | 近期（绿） |
| 30-90 | 未用（灰） |
| 90-180 | 久未打开（琥珀） |
| 180-365 | 建议清理（橙） |
| >365 | 很可能不需要（红） |

「陈旧度」= 距最近一次触达的天数，触达取 `last_opened_at`（filer 里点「打开」）> `last_reviewed_at`（点「仍需要」续期）> `filed_at` 三者最大。「状态」列对比归档时记录的文件 mtime 基线，标记「未改动/已更新/文件缺失」。

启动时如有文件超过 180 天未触达，顶部 banner 提示条数并指向回顾 tab。

每行操作：
- **打开**：在资源管理器定位 + 刷新 last_opened_at（续期）。
- **仍需要**：刷新 last_reviewed_at，重置陈旧度。
- **撤销**：移回复件箱。
- **删除文件**：删除归档文件，记录标 deleted。

> 跟踪信号是 filer 内部跟踪（点「打开」才记），不依赖 Windows atime（默认关闭、不可靠）。外部直接双击文件的打开不计入——如需覆盖所有打开，未来可接 OS 级访问监控（ETW/USN），属后续大改。

## 3. 规则引擎

内置规则按顺序匹配，首条命中胜出，`misc` 兜底。所有 `dest_template` 都是**相对 dest_root 的子路径**：

| id | 分类 | 匹配 | 子路径模板 | 最终目标示例（dest_root=D:\Filer） |
|---|---|---|---|---|
| datasheet | Datasheet | pdf + 内容识别到厂商 | `Datasheets\${vendor}` | `D:\Filer\Datasheets\ST\` |
| invoice | Receipt | 文件名含 发票/invoice/receipt/账单/收据/bill | `Receipts\${yyyy-mm}` | `D:\Filer\Receipts\2026-07\` |
| installer | Software | exe/msi/deb/dmg/appimage/pkg/apk | `Software\Installers` | `D:\Filer\Software\Installers\` |
| screenshot | Screenshot | png/jpg/jpeg/webp/bmp | `Media\Screenshots\${yyyy-mm-dd}` | `D:\Filer\Media\Screenshots\2026-07-06\` |
| document | Document | docx/doc/xlsx/.../txt/md/csv/pdf | `Documents\${yyyy-mm}` | `D:\Filer\Documents\2026-07\` |
| archive | Archive | zip/7z/rar/tar/gz/bz2/xz | `Archives\${yyyy-mm}` | `D:\Filer\Archives\2026-07\` |
| misc | Misc | 兜底 | `${yyyy-mm}` | `D:\Filer\2026-07\` |

模板变量：`${yyyy}`/`${yyyy-mm}`/`${yyyy-mm-dd}`（配置时区）；`${vendor}`（PDF 内容识别，空→`Unknown`）；`${title_or_name}`（PDF 标题或文件名 stem）；`${original_name}`；`${ext}`；`${category}`。

> PDF 走 datasheet 规则需要"内容识别到厂商"（STM32→ST、ESP32→Espressif 等）。识别不到厂商的 PDF 落到 document 规则。发票类 PDF 因文件名含"发票"先于 datasheet 命中 → Receipt。

## 4. 时区

设置里时区留空 = 系统本地时区；填 IANA 名（如 `Asia/Shanghai`、`America/New_York`）则模板日期目录按此时区生成。所有时间戳（detected_at/filed_at）按此时区存储与显示。

## 5. 冲突与判重

**同名冲突**（目标目录已有同名但内容不同的文件）：

| 策略 | 行为 |
|---|---|
| rename（默认） | 加 ` (1)`、` (2)` 后缀 |
| skip | 跳过，标 ignored |
| overwrite | 直接覆盖 |

**内容判重**（sha256 相同 = 同内容）：见上方「重复下载提示」。filer 只做下载后内容判重；下载前基于 URL/ETag 的预判重（依赖服务器给 HTTP 头，且 filer 作为文件监听器看不到 HTTP 层）未在 v1 实现，留作未来按需接 aria2/IDM 历史。

## 6. 排错

| 现象 | 原因 | 解决 |
|---|---|---|
| 收件箱不刷新 | watcher 未启动或 watch_dir 无写权限 | 设置里确认 watch_dir，重启 app |
| 「目标根目录未配置」 | dest_root 在设置里为空 | 设置里选 dest_root |
| 「请选择处理方式」 | 重复项未选 4 个动作之一 | 点重复行的某个按钮 |
| 撤销失败「归档文件已不存在」 | 归档文件被手动删除 | 记录标 missing，可删除该条 |
| 跨盘移动慢 | move 跨卷会 copy+delete | 正常，大文件需等待 |
| 扫描大目录慢 | 逐文件流式哈希+分析 | 正常，按钮显示 X/Y 进度，单文件 60s 超时自动跳过 |
| 临时文件入箱 | 后缀未被识别 | 反馈，扩充 `is_temp_filename` |

## 7. 技术栈

Tauri 2 (Rust) + React 18 + Vite 8 + TypeScript 5.6 + Tailwind 3.4。SQLite（`rusqlite` bundled）索引、`notify` 文件监听、`lopdf` PDF 元数据、`image` 图片尺寸、`kamadak-exif` EXIF、`zip` office core props + 压缩包条目、`chrono-tz` 时区。单二进制，无 sidecar、无 Python、无运行时依赖。MIT 许可证。

## 8. 其他功能

### 搜索

「搜索」tab：跨已归档文件全文检索，匹配文件名 / 分类 / 标签 / 厂商 / 元信息（EXIF/office/zip 条目）/ 路径。回车搜索，结果可打开或撤销。

### 统计

「统计」tab：已归档/收件箱总数、近 7 天归档、待清理(>180天) 计数卡；按分类、按动作的条形分布；陈旧度分层分布。

### 自动归档

设置里勾「自动归档」后，下载完成即弹归档确认小窗（预填规则建议：分类/目标/文件名/动作），用户补完标签等检索字段回车即归档。多个文件排队依次弹窗。关闭开关则新文件静默进收件箱（默认）。

### 规则编辑

设置里「规则集」可增删/上下排序/编辑每条规则的字段（id/分类/扩展名/关键词/内容匹配/目标子路径/文件名模板/动作）。保存写回 config.toml。

### 右键「用 filer 归档」

安装后，Windows 资源管理器里任意文件右键 →「用 filer 归档」→ 唤起 filer（或已运行实例）→ 自动分析入箱（若开自动归档则直接弹确认窗）。注册位置 `HKCU\Software\Classes\*\shell\filer`（仅当前用户，无需管理员；对文件夹无效）。

### 富元信息

PDF：页数/标题/厂商建议。图片：尺寸 + EXIF（拍摄时间/相机/GPS）。office（docx/xlsx/pptx）：core.xml 的标题/作者。压缩包（zip 等）：顶层条目名。全部进 sub_meta，可被搜索命中。
